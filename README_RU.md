# claude-code-powertools

CLI-инструменты, которые делают Claude Code умнее, а ваш терминал мощнее. Одна команда устанавливает и настраивает всё.

```bash
curl -sL https://raw.githubusercontent.com/dkh-ai/claude-code-powertools/main/install.sh | bash
```

Или клонируйте и запустите локально:

```bash
git clone https://github.com/dkh-ai/claude-code-powertools.git
cd claude-code-powertools
bash install.sh
```

## Что это делает

У Claude Code есть Bash-инструмент для запуска терминальных команд. По умолчанию он использует базовые Unix-утилиты — `find`, `grep`, `cat` — которые работают, но не оптимальны. Современные CLI-инструменты быстрее, дают лучший вывод и расширяют возможности Claude Code (структурные диффы, парсинг YAML, статистика кодовой базы).

Инсталлятор делает три вещи: (1) устанавливает до 15 CLI-инструментов через Homebrew, (2) настраивает shell с алиасами и интеграциями, и (3) обучает Claude Code новым инструментам, добавляя справочную таблицу в `CLAUDE.md`.

Инструменты разделены на две категории: **Для Claude Code** (8 инструментов, которые Claude использует напрямую) и **Для вас** (7 инструментов для улучшения терминала). Можно установить обе категории, одну, или выбрать отдельные инструменты.

## Каталог инструментов

### Для Claude Code

Эти инструменты Claude Code использует через Bash. Инсталлятор добавляет справочную таблицу в `CLAUDE.md`, чтобы Claude знал, когда и как их использовать.

---

#### tree

> Обзор структуры проекта

Когда Claude Code попадает в незнакомый проект, ему нужно понять структуру каталогов. `tree` даёт чистое иерархическое представление одной командой. Без него Claude делает множество вызовов `ls` или `Glob`.

```bash
tree -L 2 --dirsfirst          # Два уровня, каталоги первыми
tree -L 3 -I node_modules      # Три уровня, игнорировать node_modules
tree -P "*.ts" --prune          # Только TypeScript файлы
```

**Заменяет:** множественные вызовы `ls`/`Glob`

---

#### yq

> Парсинг и фильтрация YAML/TOML/XML

YAML повсюду — манифесты Kubernetes, конфиги CI/CD, Docker Compose. `yq` позволяет Claude Code читать, запрашивать и модифицировать структурированные данные без Python-скриптов. Синтаксис как у `jq`, но работает с YAML, TOML и XML.

```bash
yq '.services.web.ports' docker-compose.yml    # Извлечь порты
yq -i '.version = "2.0"' config.yaml           # Редактировать на месте
yq -o json config.yaml                         # Конвертировать YAML в JSON
```

**Заменяет:** Python-однострочники для парсинга YAML

---

#### shellcheck

> Статический анализ shell-скриптов

Когда Claude Code пишет или редактирует shell-скрипты, `shellcheck` ловит баги до запуска — неиспользуемые переменные, неэкранированные подстановки, проблемы совместимости с POSIX. Инсталлятор добавляет правило в `CLAUDE.md`, чтобы Claude запускал `shellcheck` на каждом скрипте.

```bash
shellcheck script.sh                # Анализ скрипта
shellcheck -e SC2086 script.sh      # Исключить конкретные предупреждения
shellcheck -f diff script.sh        # Вывод в формате diff (автоисправление)
```

**Заменяет:** ручную ревизию скриптов

---

#### fd

> Современный поиск файлов (замена find)

`fd` — быстрая и удобная альтернатива `find`. Уважает `.gitignore`, использует regex по умолчанию, имеет разумные настройки. Claude Code использует его для поиска файлов по имени, расширению или дате изменения.

```bash
fd -e py                     # Найти все Python-файлы
fd -e ts --changed-within 1d # TypeScript-файлы, изменённые сегодня
fd "test" --type f           # Файлы с "test" в имени
```

**Заменяет:** `find . -name "*.py"`

---

#### ripgrep

> Быстрый поиск по коду для Bash-пайплайнов

У Claude Code есть встроенный Grep, но `rg` (ripgrep) полезен, когда нужно объединить поиск с другими командами — через `sort`, `uniq`, `xargs` или `jq`. Самый быстрый grep, уважает `.gitignore`.

```bash
rg "TODO|FIXME" --count          # Подсчёт TODO по файлам
rg -l "import React" | xargs wc -l  # Количество строк в React-файлах
rg "api/" --json | jq '.data'    # Структурированный вывод
```

**Заменяет:** системный `grep` в сложных Bash-пайплайнах (встроенный Grep покрывает 80% задач)

---

#### scc

> Статистика кодовой базы по языкам

Когда вы спрашиваете «насколько велик проект?» или «какие языки используются?», `scc` отвечает мгновенно. Считает строки кода, комментарии и пустые строки по языкам с оценкой сложности.

```bash
scc --no-cocomo              # Статистика без оценки стоимости
scc --by-file --sort lines   # Разбивка по файлам
scc src/                     # Статистика для конкретного каталога
```

**Заменяет:** `find | wc -l`, `cloc`

---

#### difftastic

> AST-aware структурный git diff

Обычный `git diff` показывает построчные текстовые изменения. `difft` (difftastic) понимает синтаксическое дерево языка и показывает структурные изменения — перемещённые функции, переименованные переменные, переформатированный код. Инсталлятор настраивает его как инструмент диффа по умолчанию.

```bash
difft file-a.py file-b.py    # Сравнить два файла
git diff                     # Использует difft автоматически после установки
git diff --no-ext-diff       # Обойти difft, использовать обычный diff
```

**Заменяет:** построчный `git diff` (используйте `--no-ext-diff` для оригинала)

---

#### jq

> Парсинг и фильтрация JSON

JSON — универсальный язык API и конфигов. `jq` позволяет Claude Code извлекать, трансформировать и запрашивать JSON без Python. Незаменим для API-ответов, package.json и любых структурированных данных.

```bash
jq '.dependencies' package.json          # Извлечь зависимости
jq '.[] | select(.status == "active")'   # Фильтровать массивы
curl -s api.example.com | jq '.data'     # Парсить API-ответ
```

**Заменяет:** Python-однострочники для парсинга JSON

---

### Для вас

Эти инструменты улучшают ваш ежедневный опыт работы в терминале: лучшие настройки по умолчанию, умная навигация и визуальные улучшения. Инсталлятор добавляет shell-интеграции в `.zshrc`.

---

#### fzf

> Нечёткий поиск (Ctrl+R, Ctrl+T, Alt+C)

`fzf` превращает три горячие клавиши в суперсилы. **Ctrl+R** — нечёткий поиск по истории команд (больше не нужно нажимать стрелку вверх 50 раз). **Ctrl+T** — интерактивный поиск и вставка путей к файлам. **Alt+C** — переход в любой подкаталог. После первого использования вы не сможете без этого жить.

```bash
# После установки просто используйте горячие клавиши:
# Ctrl+R  →  нечёткий поиск по истории
# Ctrl+T  →  нечёткий поиск файлов
# Alt+C   →  нечёткий cd в каталоги
vim $(fzf)     # Открыть файл, выбранный интерактивно
```

**Заменяет:** стрелку вверх для истории, танец `ls` + `cd`

---

#### zoxide

> Умный cd, который запоминает каталоги

`zoxide` заменяет `cd` более умной версией, которая помнит, какие каталоги вы посещаете чаще всего. Наберите `z proj` — и он перейдёт в `~/projects` (или куда вы чаще ходите по запросу «proj»). Больше не нужно набирать полные пути.

```bash
z proj           # Перейти в самый посещаемый каталог по запросу "proj"
z doc            # Перейти в documents/docs
zi               # Интерактивный выбор с нечётким поиском
```

**Заменяет:** `cd ~/long/path/to/directory`

---

#### eza

> Современный ls с git и иконками

`eza` заменяет `ls` цветным выводом, индикаторами git-статуса и иконками файлов. Инсталлятор настраивает четыре алиаса: `ls` (базовый), `ll` (подробный + git), `lt` (дерево), `la` (скрытые файлы).

```bash
ls               # Цветной список файлов (eza)
ll               # Подробный формат с git-статусом + иконки
lt               # Дерево (2 уровня, уважает .gitignore)
la               # Показать скрытые файлы
```

**Заменяет:** обычный вывод `ls`

---

#### lazygit

> Полный git TUI

Терминальный UI для git, который делает сложные операции визуальными. Стейджинг отдельных хунков, интерактивный rebase, cherry-pick, управление стешами — всё без запоминания git-флагов. Алиас `lg`.

```bash
lg               # Запустить lazygit в текущем репозитории
```

**Заменяет:** сложные последовательности команд `git`

---

#### bat

> cat с подсветкой синтаксиса

`bat` — это `cat` с подсветкой синтаксиса, номерами строк и git-интеграцией. Инсталлятор добавляет алиас `cat` на `bat`, так что каждый просмотр файла даёт красивый подсвеченный вывод.

```bash
cat file.py      # Подсвеченный вывод (bat)
bat -A file.txt  # Показать невидимые символы
bat -l json      # Принудительная подсветка JSON
```

**Заменяет:** обычный вывод `cat`

---

#### imagemagick

> Пакетная обработка изображений

`imagemagick` даёт Claude Code (и вам) возможность изменять размер, конвертировать, обрезать и обрабатывать изображения из командной строки. Полезен для пакетной обработки скриншотов, генерации превью или конвертации форматов.

```bash
magick input.png -resize 50% output.png          # Изменить размер
magick input.jpg -quality 80 output.webp          # Конвертировать формат
magick mogrify -resize 800x600 screenshots/*.png  # Пакетное изменение размера
```

**Заменяет:** открытие графического редактора для простых операций

---

#### htop

> Интерактивный мониторинг процессов

`htop` — визуальный интерактивный менеджер процессов. Используйте, когда нужно увидеть, что потребляет CPU/память, убить зависший процесс или мониторить нагрузку.

```bash
htop             # Интерактивный просмотр процессов
```

**Заменяет:** `ps aux | grep`, Activity Monitor

## Пресеты

| Пресет | Флаг | Инструменты | Кол-во |
|--------|------|-------------|--------|
| Все | `--preset all` | Всё | 15 |
| Claude Code | `--preset claude` | tree, yq, shellcheck, fd, ripgrep, scc, difftastic, jq | 8 |
| Пользователь | `--preset user` | fzf, zoxide, eza, lazygit, bat, imagemagick, htop | 7 |
| Минимальный | `--preset minimal` | tree, shellcheck, fd, jq | 4 |

В интерактивном режиме можно набирать буквы пресетов: `a` (все), `c` (claude), `u` (user), `m` (minimal), `n` (снять всё).

## Что настраивается

### ~/.zshrc

Управляемый блок между маркерными комментариями добавляется в `.zshrc`:

```bash
# >>> claude-code-powertools >>>
# ... shell-интеграции для fzf, zoxide, bat, eza, lazygit ...
# <<< claude-code-powertools <<<
```

Все интеграции защищены проверками `command -v` — активируются только если инструмент установлен. Повторный запуск инсталлятора чисто заменяет блок.

### git config

Если вы установите difftastic, инсталлятор задаёт `git config --global diff.external difft`, делая `difft` просмотрщиком диффов по умолчанию. Используйте `git diff --no-ext-diff` для оригинального построчного диффа.

### ~/.claude/CLAUDE.md

Управляемый блок между HTML-маркерами добавляется в конец:

```markdown
<!-- claude-code-powertools:begin -->
**CLI Toolbox** (installed via brew, USE proactively):
| Tool | Use for | Instead of |
...
**Rules:**
...
<!-- claude-code-powertools:end -->
```

Это обучает Claude Code, когда и как использовать каждый инструмент. В таблицу включаются только реально установленные инструменты.

## CLI-флаги

| Флаг | Короткий | Описание |
|------|----------|----------|
| `--yes` | `-y` | Пропустить все подтверждения (авто-принятие) |
| `--dry-run` | `-n` | Превью изменений без модификации |
| `--preset NAME` | `-p NAME` | Выбрать пресет: `all`, `claude`, `user`, `minimal` |
| `--help` | `-h` | Показать справку |

### Примеры

```bash
# Интерактивный (по умолчанию) — показывает меню, позволяет переключать инструменты
bash install.sh

# Установить только инструменты для Claude Code, без вопросов
bash install.sh --preset claude --yes

# Превью изменений
bash install.sh --dry-run

# Установить всё неинтерактивно (подходит для CI/автоматизации)
bash install.sh --preset all --yes

# Установка через pipe (автоопределение отсутствия TTY, включает --yes)
curl -sL .../install.sh | bash
```

## Трекинг использования

Опционально: отслеживайте, какие powertools Claude Code реально использует. PostToolUse-хук логирует каждый вызов в JSONL-файл для последующего анализа.

### Настройка

```bash
cd claude-code-powertools
bash scripts/setup-tracking.sh
```

Скрипт копирует хук в `~/.claude/hooks/` и регистрирует его в `~/.claude/settings.json`. Запустите с `--dry-run` для превью.

### Просмотр отчёта

```bash
bash scripts/usage-report.sh                    # Текстовый отчёт (по умолчанию)
bash scripts/usage-report.sh --format markdown  # Markdown-таблица
bash scripts/usage-report.sh --format json      # Машиночитаемый формат
bash scripts/usage-report.sh --days 7           # Только последняя неделя
```

Пример вывода:

```

  ──────────────────────────────────────────

  Period: last 30 days
  Total calls: 142

  By Tool
  tree            47  ████████████████
  shellcheck      31  ██████████
  jq              22  ███████
  fd              18  ██████
  scc              9  ███
  yq               8  ██
  difft            7  ██

  By Project
  my-api                             52
  claude-code-powertools             38
  web-dashboard                      29
```

### Как это работает

Хук получает JSON от Claude Code после каждого Bash-вызова, извлекает имя бинарника и проверяет его по списку powertools. Совпавшие вызовы дописываются в `~/.claude/powertools-usage.jsonl`. Хук неблокирующий (всегда завершается с кодом 0) и требует `jq`.

## Удаление

```bash
bash uninstall.sh
```

Или удалённо:

```bash
curl -sL https://raw.githubusercontent.com/dkh-ai/claude-code-powertools/main/uninstall.sh | bash
```

Деинсталлятор:
1. Удаляет управляемый блок из `.zshrc`
2. Удаляет управляемый блок из `CLAUDE.md`
3. Откатывает `git config diff.external` если установлен на `difft`
4. Удаляет хук трекинга и его конфиг из `settings.json`
5. Предлагает `brew uninstall` для каждого инструмента (с подтверждением)
6. Предлагает восстановить бэкапы, созданные при установке

## FAQ

**Работает ли на Linux?**
Пока нет. Инсталлятор только для macOS, так как зависит от Homebrew. Все инструменты работают на Linux — устанавливайте через свой пакетный менеджер.

**Сломает ли это мой .zshrc / CLAUDE.md?**
Нет. Инсталлятор создаёт бэкапы (`.powertools-backup`) перед изменениями. Все модификации содержатся между маркерными комментариями и могут быть чисто удалены. Инсталлятор также предупреждает об обнаруженных дублях интеграций.

**Что если часть инструментов уже установлена?**
Инсталлятор определяет уже установленные инструменты и пропускает их. В интерактивном меню они отображаются как "(installed)", в сводке — как "Already present".

**Можно ли запустить повторно после добавления/удаления инструментов?**
Да, инсталлятор идемпотентен. При повторном запуске он заменяет предыдущие маркерные блоки новыми на основе текущего выбора.

**Claude Code автоматически использует эти инструменты?**
Да, как только секция CLI Toolbox появляется в `CLAUDE.md`. Claude Code читает этот файл и следует правилам — например, запускает `shellcheck` после написания скриптов или использует `tree` при изучении нового проекта.

**Что такое пресет "minimal"?**
Четыре основных инструмента: `tree` (обзор проекта), `shellcheck` (валидация скриптов), `fd` (поиск файлов) и `jq` (парсинг JSON). Они дают Claude Code максимальный прирост возможностей при минимальном размере.
